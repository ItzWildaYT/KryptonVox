What's this site about?

apparently, this site is for storing the javascript /main.js and css /main.css file that will get fetched by the client side and applied to the voxiom page.

but why? The reason the css must be fetched from the file /main.css instead of just applying on the client side, it's because there's feature called custom css style on the newest version...

let cssFile = cssCheckbox.checked ? "" : "https://kryptovox.netlify.app/main.css"; const link = document.createElement("link"); link.rel = "stylesheet"; link.href = cssFile; link.dataset.css = 'true'; document.head.appendChild(link);

if cssCheckbox is checked, that means the user is going to use their own css, that means the appended stylesheet link to the document.head is none, and append the style of their own instead. if the css code textarea is left empty, then apply the default style from voxiom.io.

The reason the js must be fetched from a file instead of just straight up applying on the client side, it's because we've made one-time apply code that lets people use our program and stay updated. so everytime the program has an update, after the updated code is being deployed to the internet, everything that fetches that file will also get updated.

document.head.appendChild(Object.assign(document.createElement('script'), { src: 'https://kryptovox.netlify.app/main.js?v=' + Date.now(), async: true }));

apply the code above this and get the newest UI version avaible and an auto-update feature.
